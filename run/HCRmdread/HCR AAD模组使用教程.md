# HCR AAD模组使用教程

## 1. 基本介绍

HCR AAD（Attack and Defense）是一个基于Minecraft Forge的模组，为游戏添加了团队攻防模式、据点管理、战术地图和兵力系统等功能，适用于PVP局域网游戏和团队游戏，本模组并未对PVP服务器进行优化，不建议在PVP服务器上使用。

## 2. 核心功能

### 2.1 据点管理

模组允许管理员创建和管理据点，据点是游戏中的关键区域，用于团队争夺，亦或者玩家各自为营。

#### 2.1.1 基本据点命令

**创建据点**
```
/hcrpi create <名称> <x1> <y1> <z1> <x2> <y2> <z2>
```
- `<名称>`：据点名称，必须为单个大写字母（A-Z）
- `<x1> <y1> <z1>`：据点的第一个角坐标
- `<x2> <y2> <z2>`：据点的第二个角坐标
- 最多可创建7个据点

**删除据点**
```
/hcrpi del <名称>
```
- `<名称>`：要删除的据点名称

**清空所有据点**
```
/hcrpi clear
```
- 清空所有已创建的据点，并同步更新所有客户端的HUD

**列出所有据点**
```
/hcrpi list
```
- 显示所有已创建的据点信息

**保存据点预设**
```
/hcrpi save <序号>
```
- `<序号>`：预设ID
- 保存当前所有据点设置为预设
- 预设文件保存在config/hcrpoints/capture_presets目录下

**加载据点预设**
```
/hcrpi load <序号>
```
- `<序号>`：预设ID
- 加载指定序号的据点预设
- 会清空当前所有据点并加载预设中的据点

### 2.2 行动攻防模式

行动攻防模式是模组的核心功能，允许管理员启动一场结构化的攻防战，每个批次包含多个据点，团队需要在短时间内完成所有据点的防守，以获得胜利。

#### 2.2.1 行动攻防命令

**创建带批次的据点**
```
/hcrpi teamfight create <批次> <名称> <x1> <y1> <z1> <x2> <y2> <z2>
```
- `<批次>`：该据点所属的批次号
- `<名称>`：据点名称，必须为单个大写字母（A-Z）
- `<x1> <y1> <z1>`：据点的第一个角坐标
- `<x2> <y2> <z2>`：据点的第二个角坐标

**列出行动模式计划据点**
```
/hcrpi teamfight list
```
- 显示所有计划中的据点信息

**删除行动模式计划据点**
```
/hcrpi teamfight del <名称>
```
- `<名称>`：要删除的计划据点名称

**清空行动模式计划据点**
```
/hcrpi teamfight clear
```
- 清空所有计划中的据点

**设置队伍角色和兵力**
```
/hcrpi teamfight teamconfig <队伍> <角色> <兵力>
```
- `<队伍>`：队伍名称
- `<角色>`：角色类型，必须是 `attacker`（进攻方）或 `defender`（防守方）
- `<兵力>`：该队伍的初始兵力值
- 每个角色只能设置一个队伍
- 暂时无法切换队伍角色，只能在创建时设置，如需要切换，请重启游戏，很遗憾，不过未来版本会考虑添加此功能。

**启动行动**
```
/hcrpi teamfight start <总批次> <结束行为>
```
- `<总批次>`：行动的总批次数量
- `<结束行为>`：当兵力耗尽时的行为
  - `terminate`：终止行动
  - `loop`：重启行动，开始新的循环
  - 理论上会按照这样子执行，但是我不好说，至少双人测试时是按照这样子执行的。

**停止行动**
```
/hcrpi teamfight stop
```
- 终止当前正在进行的行动

**进入下一批次**
```
/hcrpi teamfight nextbatch
```
- 手动进入下一批次

**保存行动预设**
```
/hcrpi teamfight save <预设ID>
```
- `<预设ID>`：预设的ID号
- 保存当前的行动设置
  - 包括队伍角色、兵力和据点信息

**加载行动预设**
```
/hcrpi teamfight load <预设ID>
```
- `<预设ID>`：要加载的预设ID号
- 加载之前保存的行动设置

### 2.3 战术地图

战术地图显示玩家位置、控制点和其他重要信息。

#### 2.3.1 地图功能
**注意**
- 如果您的屏幕未完全显示战术地图，可能需要调整屏幕分辨率或窗口大小，或者调整游戏设置中的界面尺寸来适配，当然也可以使用迷你地图，迷你地图可以单独设置尺寸。

**打开/关闭战术地图**
- 快捷键：默认M键（可在按键设置中修改）
**切换地图显示模式**
- 快捷键：默认V键（可在按键设置中修改）
**打开地图配置界面**
- 快捷键：默认X键（可在按键设置中修改）

#### 2.3.2 地图特性
- 显示玩家位置，不同队伍显示不同颜色
  - 友方玩家：绿色
  - 敌方玩家：红色
  - 未知：白色
- 显示控制点位置和状态
- 支持缩放和旋转
- 可配置显示选项

### 2.4 兵力系统

每个队伍都有兵力值，兵力会随着时间或战斗情况变化。

#### 2.4.1 兵力规则
- 进攻方和防守方各有初始兵力
- 兵力会根据控制点的状态变化
- 当一方兵力耗尽时，行动结束（根据配置的结束行为）

#### 2.4.2 低兵力警告
- 当一方兵力低于配置的阈值（默认10%）时，会播放警告音频
- 音频会在行动结束后结束

### 2.5 团队系统

模组支持基于Minecraft队伍系统。

#### 2.5.1 队伍设置
- 进攻方：通常为"ATTACKER"队伍
- 防守方：通常为"DEFENDER"队伍
- 其他队伍的玩家会被警告并建议更换队伍
- 在行动开始后发生的队伍切换不会受到警告，这也是导致您兵力不会减少的可能原因之一

### 2.6 消息弹窗系统

#### 2.6.1 发送消息弹窗
```
/hcrpi send <玩家> <边框颜色> <内容>
```
- `<玩家>`：接收消息的玩家名称
- `<边框颜色>`：弹窗边框颜色，6位十六进制代码（不带#）
- `<内容>`：要发送的消息内容

**常用颜色代码**：
- 红色：FF0000
- 橙色：FF5500
- 黄色：FFFF00
- 绿色：00FF00
- 蓝色：0000FF
- 紫色：5500FF
- 粉色：FF00FF
- 青色：00FFFF
- 白色：FFFFFF
- 灰色：888888

### 2.7 音频系统

#### 2.7.1 播放音频
```
/hcrpi playsound <音效名>
```
- `<音效名>`：要播放的音效名称
- 目前只支持：`lastStandBGM`（背水一战背景音乐）
- 建议使用WAV格式音频

## 3. 游戏规则

模组添加了以下游戏规则：

### 3.1 showPlayerLocation
- **描述**：控制是否在战术地图上显示其他玩家的位置
- **默认值**：true
- **命令**：`/hcrpi mapctrl true/false`

### 3.2 lowReinforcementThreshold
- **描述**：低兵力警告阈值，单位为百分比
- **默认值**：10
- **修改方式**：在配置文件中修改

## 4. 配置文件

模组使用Forge配置系统，分为客户端配置和服务端配置。

### 4.1 客户端配置
- **位置**：`游戏目录/config/hcrpoints-client.toml`
- **包含内容**：
  - 战术地图的显示选项
  - 音频设置
  - HUD显示选项

### 4.2 服务端配置
- **位置**：`游戏目录/config/hcrpoints-common.toml`
- **包含内容**：
  - 兵力系统设置
  - 操作模式默认值
  - 游戏规则默认值
  - 是否启用行动攻防机制

## 5. 注意事项

### 5.1 管理员注意事项

1. **队伍设置**：确保在启动行动前设置好进攻方和防守方队伍
2. **兵力平衡**：根据玩家数量和地图大小调整兵力设置
3. **结束行为**：根据游戏需求选择合适的结束行为（terminate或loop）
4. **据点名称**：据点名称必须为单个大写字母，最多7个
5. **权限管理**：所有命令都需要2级管理员权限

### 5.2 玩家注意事项

1. **队伍选择**：确保加入正确的队伍（进攻方或防守方）
2. **战术配合**：使用战术地图查看队友和敌人位置
3. **兵力意识**：关注队伍兵力情况，合理安排战术
4. **快捷键设置**：根据个人习惯设置合适的快捷键
5. **遵守规则**：遵守服务器管理员设置的规则和指令

## 6. 警告

1. **性能影响**：
   - 战术地图会占用一定的系统资源

2. **游戏平衡**：
   - 不合理的兵力设置可能导致游戏失衡
   - 建议管理员根据实际情况调整兵力参数

3. **兼容性**：
   - 模组可能与其他PVP模组存在冲突
   - 建议仅在局域网联机下使用该模组，部署在服务器上的情况并不在模组开发目标内，可能导致未知问题

4. **滥用风险**：
   - 管理员指令具有较高权限
   - 建议只授予信任的玩家管理员权限

## 7. 常见问题解答

### Q: 为什么无法使用基础据点命令？
A: 可能是行动攻防机制已启用，请使用`/hcrpi teamfight`命令

### Q: 为什么看不到其他玩家的位置？
A: 请检查：
- 战术地图是否已打开
- showPlayerLocation游戏规则是否为true
- 你和其他玩家是否在同一维度

### Q: 为什么兵力没有变化？
A: 请检查：
- 据点机制是否正常工作
- 队伍设置是否正确，这是最可能导致问题的

### Q: 为什么音频没有播放？
A: 请检查：
- 游戏音量是否开启
- fightBGM文件夹是否存在
- 音频文件是否正确放置
- 低兵力阈值是否设置合理
- 音频文件并不会从服务端向客户端同步，因为大多数时候音频文件十分巨大，对服务器是个不小的负担，因此需要你自己寻找音频并放入文件夹。

### Q: 如何修改快捷键？
A: 进入游戏设置 -> 按键控制 -> 在"hcrpoints"分类下修改对应快捷键

### Q: 为什么提示权限不足？
A: 所有管理员命令都需要2级管理员权限
